## БЛОК 2: МЕТОДОЛОГИЯ GSD (Spec-First)

### Принцип: Спецификация -> Код -> Тесты -> Деплой

Каждая задача проходит 7 фаз:
1. **SPEC** -- написать спецификацию в формате IL (Instruction Ledger)
2. **DESIGN** -- ArchiMate / C4 диаграмма если нужна архитектура
3. **IMPLEMENT** -- код строго по спецификации
4. **TEST** -- pytest + ruff + semgrep (минимум 80% coverage)
5. **REVIEW** -- автоматический code review агентом-ревьюером
6. **DEPLOY** -- rsync/git push по правилам одобрения
7. **CLOSE** -- закрытие IL-записи, обновление COMPLIANCE-MATRIX

### Instruction Ledger (IL) формат:
```
IL-XXX | Статус | Приоритет | Описание | Блокировки | Дата
```
Статусы: OPEN -> IN_PROGRESS -> REVIEW -> DONE | BLOCKED

## БЛОК 4: АГЕНТ-КОНТРОЛЁР ИСПОЛНЕНИЯ

### Роль: Supervisor Agent
Агент-контролёр запускается автоматически при старте сессии и:

1. **Проверяет прогресс** -- сканирует INSTRUCTION-LEDGER.md
2. **Находит незавершённые задачи** -- статус OPEN или IN_PROGRESS
3. **Предлагает план продолжения** -- формирует список действий
4. **Контролирует исполнение** -- проверяет что каждая фаза GSD выполнена
5. **Возвращает к решению** -- если разработчик остановился, контролёр напоминает

### Алгоритм контролёра при старте:
```
1. Прочитать INSTRUCTION-LEDGER.md
2. Найти задачи со статусом != DONE
3. Отсортировать по приоритету (P0 > P1 > P2)
4. Для каждой задачи определить текущую фазу GSD
5. Предложить CEO: "Продолжаем [IL-XXX]? Текущая фаза: [ФАЗА]. План: [ДЕЙСТВИЯ]"
6. При получении "да" / "продолжай" -- выполнять без дополнительных вопросов
7. При завершении задачи -- перейти к следующей по приоритету
```

### Самовозврат к работе:
Если разработчик "завис" или сессия прервалась, при следующем запуске:
- Контролёр АВТОМАТИЧЕСКИ определяет точку остановки
- Предлагает: "Последняя задача: IL-XXX, фаза: [X]. Продолжить?"
- НЕ ЖДЁТ повторного объяснения контекста от CEO

## БЛОК 7: ЧЕК-ЛИСТ САМОПРОВЕРКИ (Controller Agent)

При завершении КАЖДОЙ задачи контролёр проверяет:

- [ ] IL-запись обновлена (статус = DONE)
- [ ] Тесты пройдены (pytest)
- [ ] Линтинг чист (ruff)
- [ ] Security check (semgrep)
- [ ] Код закоммичен и запушен
- [ ] COMPLIANCE-MATRIX обновлена
- [ ] Документация обновлена (если нужно)
- [ ] Следующая задача определена

Если хоть один пункт не выполнен -- задача НЕ считается закрытой.
